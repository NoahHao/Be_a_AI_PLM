
# 客户交流

    在昨天晚上，想到自己的to do list上，或者说本书的大纲上还可以加上一条，那就是 
    【 】与业界有经验的 AI产品管理专家交流，并总结访谈记录。
    这一条有难度，除非文章起到一定的影响力，否则很难触达这些业界专家，只能与身边的人聊。

    今天觉得可以从客户交流这一条开始写起，因为在我梳理自己的工作经验时，结合当前AI Agent逐渐在各个领域取代了不同的工作角色，其中以 Claude code取代程序员这一条收效最好。这听起来可能有点讽刺，对于开发这个产品的程序员来说，听起来有点悲壮和浪漫。基于当前的这个趋势，我也在思考一个问题，我的哪些工作是AI Agent比较难以取代的。 

    我给出的答案是，客户交流。 所有流程化的东西都容易被Workflow所固定，所有能快速被验证结果的事，也容易被大模型固定能力。但是客户交流这一点不同，这是人和人之间的交互。

    我在近3年国外飞了4个大洲，多个国家，比较忙碌的一年，在7个月飞行了20W公里，仅在大型展会主讲过的高层交流就已经有40多个。其中我遇到过一件印象深刻的事儿，在非洲某个国家与客户进行交流的时候，现场有一个产品经理。在我对方案进行基础的讲解之后，大家开始展开了讨论，现场有一个一线用法语和客户交流，我在现场用英语和客户交流。当我们遇到一些关键的讨论需要记录下来的时候，我们就对这个产品经理进行传递，让他把信息记录下来，当时客户意味深长的一笑，然后用手拿起这个客户经理放在桌子上的翻译器，用英语说了一句 
        “他当然记录下来了”
    把这个翻译器在手里朝我们摇了摇，然后又放到了桌子上。

    我当时脑子里只有一个想法：“ 如果我到了一个新的国家，必须要用一门新的语言作为工作语言，玩命也要把这门语言搞定，绝对不要被这样轻视”

    这绝对不仅仅是一门语言的问题....

    ### 客户分类
     在我工作中经常遇到的客户角色主要有这些 ： 
     - CXO ： 如果是初次高层对标，通常都是通过本次解决方案的介绍，带动新产品和新解决方案在客户侧落地。如果客户本身已经采购了我所负责的产品及解决方案，那么主要就是要通过高层对标，持续体现对客户的重视程度，闭环客户当前在产品使用中存在的问题，收集客户的需求，如果客户技术水平比较高，对产品及竞分产品有深刻的理解，总结来说，就是客户的需求比较多，比较细，我们的开发人力又不足以做完客户的所有需求，那么此时就需要给客户讲清楚我们的产品路标roadmap，也就是我们接下来的几个版本里 ，都做了什么，每个不同版本的定位是什么，这个版本定位是如何贴近了客户的长期战略发展。让客户安心，我们的产品线在公司处于怎样的定位，但结论是一样的，那就是，

     我们的版本会长期演进，客户的需求我们非常关注，并且会逐步落实，我们的产品有足够的竞争力，并且持续向行业第一的方向去演进。
     对于不能完成的需求，要说清楚，这个需求为什么不是核心需求，这个需求因为什么缺乏使用场景，最好的方向是说明，这个技术特性已经不符合当前的方向了，我们正在规划一个颠覆性的技术，在什么时间能够为客户交付。 
     
     能够聊到产品路标的，已经是我们深度合作，关系非常良好的客户了，这种通常还会在一些关键场景进行识别，并协同规划一个联合创新，把某个认为能够在未来批量复制的特性，在客户侧进行试点使用。

     这些说的都是产品能力的部分，可以简单的归纳一下这部分交流材料的构成。 交流材料通常也都是由产品管理来编写，或者是牵头编写的。我们可以理解成， 因为产品经理对客户的满意度负责，那么自然要对交流的满意度负责，对材料负责自然成为了关键的一环。

     对于高层交流的材料，一般分为， 
     1.客户交流回顾，当前项目进展，客户战略对齐。 
     2.行业及产业趋势洞察。 
     3.解决方案介绍，这部分一般是从宏观到微观。从近期我们取得的行业认可，一般就是一些证书开始。然后到产品层，先介绍整体解决方案，然后总结出几个解决方案关键的竞争力大类。 按照顺序，将这些竞争力大类分别拆解到各个子产品上，这里要体现人无我有，最差也是人有我优的竞争力点。 这里能够引入一个新的产品管理工作，就是 竞争分析， 标记为一个to do list ，在后面进行补齐，需要注意的是，竞争分析本身就是需求分析的一部分，或者说，前置与竞争力规划之前。
      【 】竞争分析方法的介绍。 至少包括IDC洞察产品排行，主力厂家，对位产品，领军产品的核心竞争力等。。。

     4.最佳实践，列出一些我们通过这个解决方案突破的客户。这里主要是打消一些客户的顾虑，尤其是第一次交流未来计划突破的客户，客户是不希望做小白鼠的。 中小客户一般会关注所在区域，最好是这个国家头部客户的解决方案，最好还能安排一下两个客户的对标，或者参观，这样基本就能够推动落单。

     这些还会引出一个环节上的任务，也会纳入到我的to do list中，就是 客户站台材料。 对于一个长期合作的客户，最好是有行业影响力的，我们会需要这个客户为我们在大型展会上为我们站台，那此时就要为客户写这个站台材料，需要控制好这个度，因为客户到国外开会，不是给你干活的，一方面对方是借助这个平台来宣传下自己公司和自己，另一方面我们又需要把自己的解决方案放进去，所以要和这个CXO下面的员工持续沟通，多次刷这个材料，一时间我还说不清楚这个度，我能感受到的就是，不要聊太多细节，聊一些行业趋势和趋势如何匹配了这个解决方案。这部分工作整体算是营销的一环，我们又能在产品管理的流程环节补充上关键的一环，那就是上市营销。

     需求分析 -> ...  -> 上市营销。

     回归到高层交流的主线任务，这里的角色不仅仅是CXO这么简单。 我们会遇到周边角色及诸多限制。 这里还要引出一个关键的思想，就是

     --Rule  思考问题时永远带上 限制条件  --Rule  我们的工作，就是在基于现状，发现限制，并基于限制开始工作。 谁不知道要把手机做到苹果那样？把大模型能力做到ChatGPT 5.X这样？把Agent做的OpenClaw这样？ 问题就在于我们是在夹板中工作，竞争力规划的太大，就说明产品管理对自己的产品和组织缺乏认知，汇报的时候都不是被骂回来这么简单，对这个人的定位就成了  “外行”。

     谁在环节中，谁就会给我们带来限制。 高层交流的组织是由一线发起，由客户经理和产品经理进行沟通，一线的领导会全程跟进，并陪伴。一个客户关系能够做到这个份上的客户，本身就是市场的重要粮仓，是不容有失的。 同时对于产品线来说，这也是领导们的重要任务，影响他们升官发财的。所以材料不可避免的就要被这些人一遍又一遍的评审，行话叫  Dry Run ，一般就是评审一下材料逻辑，演讲者的语言能力和风格，不同国家的客户，不同年龄的客户，自然有不同的风格。 客户经理通常会提供一份客户的报告，介绍一下这个CXO的年龄，个人背景等，便于我们平衡材料的技术占比，展示风格等。

     在我前面举例中，我提到了一个语言不行的一线，并说，客户不满意的点未必就是语言。 语言是基础，没有自然不行，但有些时候，交流的效果还真就不在材料的好坏，技术的深浅，我个人觉得是一种风格。 着装自然是一点，区别场合来平衡自己着装的正式程度，我个人会专门选一些日常就能穿的正装，也就是英伦的西装，鞋子选哪种像皮鞋的鞋子，中和一下正式感。其实穿衣服这里就有很多经验，比如参加展会的时候，最好穿大半码的皮鞋，因为一天下来，自己的脚会越来越肿，如果鞋子正好，那么会导致你特别疲惫。

     

     关于风格，我还没有总结出来要如何培养。 我自己比较人来疯，人越多越high，效果越好，人少了反而有点没感觉。很多人比较担心客户挑战自己，也就是问一些问题自己回答不上。 我们肯定不能做到技术专家一样任何细节都能回答好，但是我个人通常没有太大压力，我是很喜欢交流的，我之前在公司内部的个人推优，介绍自己的亮点工作时，我也提到了一点，就是 要交流而不是展示， 传达的最好方式就是交流，自己哇哇一顿说，其实对方听没听都不知道，大部分讲解者，我认为水平一般，其实想说的就是，这种人讲讲固定材料还行，但也就到这步了，不饿能和客户交流的，也就是不能把客户的东西带回来。 如果我们只看产品管理的工作要求的话，把底层需求带回来才是我们的核心工作。 当然一线肯定不会认可这个点，他希望你最好只介绍方案，而不给客户带来任何负担，除非能够为他未来拓展项目带来优势。 说了这么多，希望你能体会到这份工作，在一个固定流程里是如何平衡各方面诉求，并把自己的任务无声无息完成的。

     另外还值得一提的是，CXO交流的衍生任务，或者说另一种形式，就是陪餐 。 我接触比较多的是国外客户的陪餐，国内的基本没接触过，听说对酒量有一定要求。。。 这里我想传递的一点信息就是，客户关系的维护，或者说我们讲解的效果是不是好，有没有给客户留下印象，其实和我们讲的主线任务没啥关系。 我这里有个比较经典的案例

     当时是在北非的一次交流，中午陪餐的时候是一个头发都白了的技术专家，说法语的，对方能力更广泛，那就向下兼容我，说英语。 选酒的时候，对方友善的把酒单给我，让我去选，
     我说道，“北非的红酒很出名啊，周末我们在超市买酒的时候，不知道买什么酒合适，有一个友善的男士，同样也是顾客，给我们聊了很长时间，给我们推荐的几款酒都很好喝。 我个人很喜欢喝酒，走到哪儿都会尝尝当地的酒，您要是对酒有研究的话，那就帮忙给我们挑一款”

     对方挑了一款酒后，顺便问到，我们周末的时候还干什么了，我说：“我们去周围最近的一个Safari看了看动物。体验确实很新颖，第一次这么近的去看动物。”  对方拿起手机给我们介绍他在一些国家打猎的照片，我们又围绕着生活聊了很多。。。

    从头到位，我们没聊任何一条与解决方案及技术有关的事情。

    事情发生在第二天，展会的最后一天，我讲解完后。这位老先生过来了一趟，给我展示了一下今天新采购我们解决方案的一个客户，说到：“今天展会结束就回去了，再来看看你”

    我个人觉得这也是一次成功的交流。还有一些情况就属于误打误撞了，比如一次我穿的也不是很正式，主要是脚下瞪著一双国内买的AJ，当天客户的主C刚好就是一个鞋子的收藏者，问了我一大堆关于这个球鞋的事儿。。。 我想说的是，这至少把交流的气氛奠定了一种友善的基调，交流效果不可能不好。

    ## 心理建设
    写到这里的时候，我已经有点觉得自己有点臭不要脸了，而且发现写的很多东西比较难复刻，不是说给一个操作建议，照着这个指导来，就搞得定的。比如这个莫名其妙的心理建设。其实我自己是不需要做啥心理建设的，我一般不怎么爱说话，但是一到需要讲东西的时候，就有点请神上身的感觉，而且跟人在哪个国家没啥关系。我从小到大大部分的时间都是自己一个人待着，不过从高中毕业开始就开始做家教了，大学里的时候也一直做这个，甚至在在线教育没变成固定工资的时候，直播讲过几次，完全不需要心理建设，直接就开讲。 我大二的时候，去日本短期交换过一段，过程中会有一个活动，日本学校里的一些“主管”（我不知道该如何描述）来参加一下这个活动，那自然我们就需要做一些发言和交流，我的日语水平是比较差的，但是最后听到的评价是“虽然语言不怎么样，但是效果是最好的，很自然” 。 在海外做高层交流，或者内部做培训的时候，对方的年龄，职级，社会地位肯定大部分都远高于我，最早和某国家第一大银行CXO交流的时候我28岁，在30岁的时候和x国第一大银行的CXO组织了一个为期2天的交流，这个客户对整个产品线都非常重要，不仅仅是我们这一个子产品，因为他能和产品线总裁直接建立链接，所以很多中层主管特别看重这个客户，这个项目，道理很简单，直接影响到他们的升官发财。 我负责这个交流的时候，依旧没什么心理负担还是正常去讲，效果也还不错。

    其实我完全可以在这里扯一些，多练练就好了，这样的废话，实际上锻炼的机会本身就比较难得，如果不是因为疫情期间海外人少，一些老资历不愿意过去，我在年轻的时候是很难有这样的机会去进行这样的交流的，毕竟一次交流的成本太高了，客户没有想象的那么好约。 练得多了有没有用，可能有，但我觉得还是和个人的性格，和心理底色有关，我没遇到别人攻击我的交流水平，也可能是有人攻击过，但是我心理素质太好，完全不往心里去。 如果非让我找出唯一的一个，能算的上强项的个人能力，我就会提到我的交流，就是这么坚定。

    就在昨天，下午2点就要交流，我在上午11点才接到这个通知，整份材料我没看过几次，简单的听过一遍视频介绍，下午就直接开讲，对面有 安全主管，数通主管，存储主管，分类特别细致，但我还是把这个交流成功的搞定了。在听到这个诉求传递过来的时候，我就觉得，我肯定能搞好，不管时间有多短。 

    说到时间短，当年有一次情况比较变态，我刚从国外回来，人还在酒店待着，突然听到一个诉求，要在3天后到达非洲什么国家支持一个展会，我当时正在吃饭，听到消息后直接就收拾了一下衣服，连个行李箱都没有，直奔海外了，就是这么狠，最后在本地靠同时的纸币和充电宝度过了一周，因为我连个转换头都没有。。。
    还有一次交流，第一次支持一个领导，这个领导在未来的一年里都是我的领导，当场就让我去讲这个材料，当时客户已经坐在对面了，我甚至还“跃跃欲试”了一下，看了第一页直接说：“领导，这个真搞不定，我第一次看这个材料，刚回国不久”，这次交流是这个领导讲的，在第二天，我就接手了这个材料的讲解了.

    说实话,这几天我在准备找新的工作,心中诸多忐忑,担心很多事情,写了这么多后,某种情况下建立了自己的自信心,总结,梳理一下历史工作后,有了逻辑才会在未来的面试中描述的更体系化,总是没有害处的.
